# Guía de Entrega y Venta a Cliente (Manual del Webmaster)

Esta guía te explica paso a paso el proceso profesional para vender, entregar y mantener el sitio web para tu cliente (el contador), y cómo posicionarte como su asesor tecnológico (Webmaster).

## 1. El Rol del "Webmaster" Hoy
Antiguamente se decía "Webmaster". Hoy los términos más modernos son:
*   **Desarrollador Web (Web Developer):** Si construyes el sitio.
*   **Administrador del Sitio (Site Manager):** Si lo mantienes actualizado.
*   **Consultor Digital:** Si le ayudas con correos, mapas y redes.

Tu objetivo es ser un **Partner Tecnológico**. No solo le vendes un producto (la web), le vendes un servicio continuo (tranquilidad y soporte).

## 2. Modelos de Negocio
Tienes dos formas principales de cobrar esto:

### A. Pago Único + Entrega Total (Hand-off)
*   **Cobras:** Un monto alto una sola vez (ej: $200.000 - $500.000+ CLP dependiendo de la complejidad).
*   **Entregas:** Todo el código, las claves del hosting y el dominio.
*   **Desventaja:** Si el cliente rompe algo, te llamará gratis para arreglarlo. Pierdes el contacto mensual.

### B. Pago Inicial + Mantenimiento Mensual (Recomendado) ✅
*   **Cobras:** Un monto inicial por el desarrollo (más bajo) y una mensualidad (fee) por "Soporte y Mantenimiento" (ej: 1 UF/mes).
*   **Incluye:** 
    *   Mantener el sitio online (Hosting).
    *   Renovar el dominio (costo anual, se lo cobras a él).
    *   Pequeños cambios (cambiar un teléfono, una foto, una fecha).
    *   Soporte técnico si algo falla.
*   **Ventaja:** Generas un ingreso pasivo y fidelizas al cliente. Tu cliente no quiere aprender GitHub, quiere que tú te encargues.

## 3. El Proceso de Entrega (Paso a Paso)

### Paso 1: Validación Final (Quality Assurance)
Antes de mostrarlo, revisa tu propia checklist:
- [ ] ¿Funcionan todos los enlaces del menú?
- [ ] ¿El formulario de contacto envía correos de verdad? (Pruébalo).
- [ ] ¿Se ve bien en el celular de tu mamá/amigo? (Prueba en móviles reales).
- [ ] ¿Están actualizados los datos de contacto (teléfono, dirección)?
- [ ] ¿Tiene el favicon (ícono de la pestaña)?

### Paso 2: Compra del Dominio (.cl)
Esto es lo único que **TIENE** que estar a nombre del cliente legalmente, pero tú puedes hacer el trámite.
1.  Entra a **NIC.cl**.
2.  Crea una cuenta a nombre del cliente (o pídele sus datos).
3.  Compra el dominio (ej: `contabilidadcmp.cl`).
4.  **Tip Pro:** Si ofreces el servicio mensual, tú te encargas de configurar los DNS (apuntarlos a GitHub Pages) para que él no se estrese.

### Paso 3: Publicación (Deploy)
Sigue nuestra `GUIA_PUBLICAR_WEB.md`.
1.  Sube la versión final a GitHub.
2.  Activa GitHub Pages.
3.  Conecta el dominio `.cl` en GitHub Settings.
4.  Espera que active el candado verde (SSL/HTTPS).

### Paso 4: La Reunión de Entrega
No solo le mandes un WhatsApp. Haz una videollamada o reunión.
*   **Muestra la web:** Navega por ella, muestra cómo se ve en el celular.
*   **Prueba en vivo:** Llena el formulario de contacto junto a él y que vea que le llega el correo. ¡Eso vende confianza!
*   **Entregables:** Envíale un correo formal con:
    *   Link de la web.
    *   Accesos (si aplica).
    *   Los manuales que ya creaste (Guía de Correo, Mapas, etc.).

## 4. Servicio Post-Venta (Mantenimiento)
Aquí es donde te transformas en su Webmaster. Ofrécele un plan mensual que incluya:
1.  **Respaldo:** "Yo tengo el código seguro en la nube".
2.  **Actualizaciones:** "Si cambias de número o dirección, me avisas y yo lo cambio en el día".
3.  **Monitoreo:** "Yo reviso que la página no se caiga".

## Resumen para el Cliente
*"Estimado Cliente, su página ya está operativa. Usted es dueño del dominio (la marca), y yo me encargo del alojamiento y la parte técnica para que funcione 24/7. Aquí le dejo los manuales de uso para sus correos y redes sociales."*
